# 输出格式

本文件定义文章最终输出的格式规范。

## 文件

- 文件名：基于文章标题，`.md` 后缀
- 位置：项目根目录（`/Users/dong/Projects/snowboat-blog/`）
- 不覆盖现有同名文件；如果存在，追加序号或询问用户

## 文章末尾必须按顺序包含三节

1. `## 作者其它文章`
2. `## 本文参考文献`
3. `## 附录：原始草稿`

节之间用 `---` 分隔。

### 1. 作者其它文章

列出作者已发表的其它文章，按**从新到旧**排列（最新文章在最上面），用 X 帖子链接。

**不包含当前文章本身**。

数据源：读取项目根目录的 `README.md`，从 "Articles" 表格提取除当前文章外的条目，用其中的 X Post 链接。跳过没有 X Post 链接（`—`）的条目。

格式：`- [文章标题](X帖子URL)`

示例：

```markdown
## 作者其它文章

- [学物理的八方英雄们，物理学已死，请转行搞AI](https://x.com/snowboat84/status/2044584627046920278)
- [不会编程、没有融资、没有员工，他怎么一个人做到年入2000万](https://x.com/snowboat84/status/2044216044575998136)
```

### 2. 本文参考文献

- 列出所有引用的来源
- 使用 Markdown 超链接格式：`[标题](URL) - 来源`
- 每条独立一行
- 顺序按正文引用顺序，或按重要性

**注意节名是"本文参考文献"，不是"参考文献"**。

示例：

```markdown
## 本文参考文献

- [Pieter Levels 个人网站](https://levels.io) - 自述
- [NASA Artemis II Mission Overview](https://www.nasa.gov/artemis-ii) - NASA 官方
```

### 3. 附录：原始草稿

- 将用户的原始草稿以引用格式（`>` ）附在最后
- **不要修改原始草稿的任何内容**，包括其中的破折号、错别字、不通顺的表述
- 标题格式：`## 附录：原始草稿`

示例：

```markdown
## 附录：原始草稿

> 这是用户的原始笔记。
>
> 可能包含破折号——这里也不改。
```

## 完成后告知

向用户返回：

1. 文件完整路径
2. 文章结构（主节标题清单）
3. 字数（大致）
4. 本文参考文献条数
5. 作者其它文章条数
